**版本**：v0.4

**状态**：草稿

## 一、产品概述

### 1.1 产品背景

当前市面上的穿搭类应用（Stylebook、Acloset、搭搭等）核心能力停留在「搭配清单推荐」层面，无法让用户直观看到自己穿上某套衣服的实际效果。用户每天面对衣橱时仍然陷入「有很多衣服却不知道穿什么」的困境。

AIGC 图像生成技术（虚拟试穿模型）的成熟，为这个问题提供了全新的解决路径。

### 1.2 产品定位

> **一个能「看到自己穿上效果」的 AI 每日穿搭助手**

用户上传自己的衣物和个人照片，App 通过 AIGC 生成用户真实穿着任意搭配组合的效果图，并结合每日天气和出行场景，智能推荐当天最合适的穿搭方案。

### 1.3 核心价值主张

- **所见即所得**：不是「这件配那件」的文字推荐，而是生成你穿上效果的真实图像
- **懂你的衣橱**：归档你真实拥有的衣物，而非通用搭配模板
- **每日新鲜感**：结合天气 + 场景，每天打开都有专属推荐

## 二、目标用户

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| **年龄** | 18–35 岁 |
| **性别** | 以女性为主，兼顾男性 |
| **特征** | 有一定衣物积累，有穿搭意识，但日常选衣耗时长 |
| **设备** | iOS / Android 智能手机 |
| **场景** | 前一天晚上提前准备穿搭、当天早晨出门前临时调整、购物前规划搭配、记录当日 OOTD |

### 2.2 用户痛点

1. **选择困难**：衣橱里衣服不少，每天早上却不知道穿什么
2. **试穿成本高**：想知道搭配效果必须真实穿上，费时费力
3. **缺乏灵感**：翻来覆去就那几套，忘记自己有哪些衣服
4. **场景错配**：穿了不合时宜的衣服（天气/场合判断失误）

### 2.3 用户旅程

```
场景 A：前一天晚上（主要场景）
  → 打开 App，系统自动识别「为明天准备」
  → 天气卡片显示明日天气预报（温度范围、天气状况、穿衣建议）
  → 浏览 3 套 AI 生成的穿搭效果图（自己穿着自己衣服）
  → 选择一套，心中有数，安心入睡

场景 B：当天早晨（临时调整）
  → 打开 App，系统自动切换为「今日穿搭」
  → 天气卡片显示今日天气预报
  → 确认昨晚选好的搭配，或根据实际天气微调
  → 出门
  → 可选：拍照打卡 OOTD，记录当日穿搭

场景 C：新用户首次体验（穿搭广场）
  → 首次打开 App，进入「穿搭广场」
  → 浏览他人分享的精美穿搭效果图
  → 看到喜欢的搭配，点击「试穿同款」
  → 上传自己的人像，AI 生成试穿效果图（Aha 时刻！）
  → 点击「加入衣橱」收录喜欢的单品
  → 自然过渡到建立自己的衣橱
```

## 三、产品主链路

```
【Step 1】建立衣橱
  用户拍照 / 上传单品图片
      ↓
  AI 自动抠图（去背景）
      ↓
  AI 识别分类（上衣 / 下装 / 外套 / 鞋子 / 配饰）
      ↓
  自动打标（颜色 / 风格 / 季节 / 场合）
      ↓
  单品入库，衣橱归档完成

【Step 2】建立个人形象
  用户上传全身人像照片（正面，光线清晰）
      ↓
  即时保存原图，后台异步触发 AI 处理（用户无需等待）
      ↓
  AI 分析人像特征（Qwen-VL-Max：性别、体型、肤色、发型、气质风格）
      ↓
  AI 人像优化（qwen-image-edit-max，1536×2048 高清输出）：
    - 自然光影均匀化（禁止磨皮/美白等过度美颜）
    - 构图优化（人物占比过小时裁剪放大）
    - 背景干扰元素去除（路人、杂物，保留整体场景）
    - 低分辨率图像超分辨率提升
      ↓
  结构化描述存入 DB，美化后图片作为后续试穿基准
      ↓
  试穿/推荐优先使用美化后人像，未处理完时降级使用原图

【Step 3】AIGC 穿搭生成
  从衣橱中选取单品组合（手动 or AI 推荐）
      ↓
  调用虚拟试穿模型（DashScope qwen-image-2.0-pro）
      ↓
  生成用户穿上该组合的真实效果图（同一组合自动缓存，不重复生成）
      ↓
  展示给用户，支持重新生成 / 微调

【Step 4】每日推荐
  每日打开 App，选择人像
      ↓
  LLM（Qwen-Max）分析衣橱单品，智能匹配 5 组搭配
      ↓
  批量调用试穿模型，生成每组搭配的效果图
      ↓
  多模态 LLM（Qwen-VL-Max）对每套效果图进行五维评分
      ↓
  按综合评分排序，取 Top 3 展示「今日穿搭推荐」
```

### 4.2 P0 功能详细需求

#### 4.2.1 衣橱管理

**单品上传**
- 支持拍照上传（调用系统相机）
- 支持从相册批量选择上传（最多 20 张/次）
- 上传后 AI 自动去背景，输出透明底单品图
- AI 自动识别并填写：分类、主色调、风格、适合季节
- 用户可手动修改所有自动识别字段
- 支持录入：品牌、购入价格、购入日期、备注
- 重复上传检测：保存原始图片并计算 SHA-256 哈希，上传时自动比对，发现重复则弹窗提示已有单品信息，用户可选择取消或仍然添加

**单品分类体系**
```
服装
  ├── 上衣（T恤 / 衬衫 / 针织 / 卫衣 / 背心）
  ├── 下装（裤子 / 裙子 / 短裤）
  ├── 外套（夹克 / 大衣 / 风衣 / 西装）
  └── 连体（连衣裙 / 连体裤 / 套装）
鞋子
  ├── 运动鞋 / 板鞋
  ├── 皮鞋 / 正装鞋
  ├── 凉鞋 / 拖鞋
  └── 靴子
配饰
  ├── 包袋
  ├── 帽子
  ├── 围巾 / 披肩
  └── 首饰 / 腰带 / 眼镜
```

**衣橱管理操作**
- 支持按分类、颜色、季节、场合多维筛选
- 支持搜索单品（关键词 / 品牌）
- 支持标记「闲置」「借出」「捐赠」状态
- 支持删除单品（二次确认）

#### 4.2.2 个人形象

- 用户上传全身正面照片（引导提示：光线均匀、背景简洁、全身入镜）
- 上传后即时保存，后台异步触发 AI 处理（用户无需等待，可立即使用原图试穿）
- AI 自动分析人像特征（性别、体型、肤色、发型、气质风格），生成结构化描述
- AI 人像优化（qwen-image-edit-max，1536×2048 高清输出）：
  - 自然光影均匀化，保留皮肤纹理和面部细节（禁止磨皮/瘦脸/美白等过度美颜）
  - 构图优化：人物占比过小时自动裁剪放大
  - 背景处理：去除路人、杂物等干扰元素，保留整体场景氛围
  - 低分辨率图像超分辨率提升
- 美化后图片作为后续试穿和推荐的基准人像（未处理完时降级使用原图）
- 人像描述注入匹配推荐 prompt，实现个性化搭配建议
- 支持上传多张人像（不同姿态 / 场景备用）
- 设置「默认人像」，用于每日推荐生成
- 人像仅存储在用户本地 / 私有云，不对外展示，不用于模型训练

#### 4.2.3 AIGC 试穿生成

**触发方式**
- 用户在衣橱中选择单品后，点击「试穿」
- 用户在推荐页选择搭配方案后，系统自动生成效果图

**生成参数**
- 基准人像：使用用户设置的默认人像
- 单品组合：上衣 + 下装（必选），外套 / 鞋子 / 配饰（可选）
- 生成数量：每次生成 1 张，可点击「重新生成」

**生成体验**
- 生成等待时间目标：≤ 15 秒
- 等待期间展示加载动画，不可进行其他操作（避免重复请求）
- 生成失败时给出明确提示并提供重试入口

**图片处理**
- 生成图默认保存至「我的搭配」收藏夹
- 支持保存到系统相册
- 支持分享（微信、小红书等）

**缓存机制**
- 同一搭配组合（人像 + 上衣 + 下装）只生成一次，结果自动缓存
- 再次请求相同组合时直接返回缓存图片，避免重复调用 API

**技术方案（MVP 阶段）**
- 调用 DashScope qwen-image-2.0-pro 模型（阿里云百炼平台）
- 生成分辨率：768×1152 px
- 备选：OOTDiffusion / IDM-VTON 自托管（成本更低，需 GPU 资源）

#### 4.2.4 每日推荐

**触发时机**
- 用户进入推荐页后选择人像，点击生成
- 用户可手动刷新推荐（重新生成 / 仅重新打分）

**推荐流程（MVP 已实现 — LLM 驱动 Pipeline）**

```
输入：
  - 用户衣橱中的所有单品（含颜色、风格、季节、场合等属性标签）
  - 用户选择的人像
  - 目标日期的天气预报（温度范围、天气状况、穿衣建议）

Step 1 — LLM 智能匹配（Qwen-Max）
  将衣橱单品列表（ID + 属性）+ 目标日期天气预报发送给 LLM
  LLM 基于颜色协调、风格统一、天气适宜、多样性原则，输出 5 组最佳搭配组合

Step 2 — 批量虚拟试穿（qwen-image-2.0-pro）
  为每组搭配生成穿搭效果图
  已有缓存的组合直接复用，不重复调用

Step 3 — AI 多维评分（Qwen-VL-Max）
  对每套效果图进行五维度独立评分（见 4.2.5）
  综合分 = 五维均值

Step 4 — 排序展示
  按综合评分降序排列，取 Top 3 作为当日推荐
  保存到 DailyRecommendation 表

输出：
  - 3 套搭配方案的虚拟试穿效果图
  - 每套方案的五维雷达图评分 + AI 搭配评语
  - 搭配理由（来自 Step 1 LLM 输出）
```

**实时进度（SSE 流式推送）**
- 整个 pipeline 通过 Server-Sent Events 实时推送进度
- 前端展示当前步骤状态（匹配中 → 生成中 → 评分中 → 完成）

**重新打分**
- 支持复用已有穿搭效果图，仅重新调用 AI 评分
- 适用于调试 prompt 或更换评分模型后快速验证

**天气 / 场景联动（已实现天气联动 + 目标日期智能选择）**
- 接入和风天气 API（3 天预报 + 穿衣指数）
- 城市选择：支持搜索全国 900+ 城市（本地静态数据，零 API 调用）
- **目标日期智能选择**：
  - 系统根据当前时间自动判断：中午 12:00 前 → 目标为「今天」，12:00 后 → 目标为「明天」
  - 用户可手动切换目标日期（今天 / 明天 / 后天）
  - 天气卡片显示目标日期的**预报数据**（温度范围、天气、穿衣建议），而非实时天气
  - 推荐页标题随目标日期动态变化（「今日穿搭推荐」/「明日穿搭推荐」）
  - LLM 匹配 prompt 使用目标日期的预报作为天气上下文
- **天气数据缓存**：按城市+小时缓存到 SQLite（WeatherCache 表），同城市同小时内只调一次 QWeather API，30 天自动过期
- 场景选项（已实现）：约会甜蜜、职场通勤、闺蜜聚会、周末逛街、健身运动、正式场合
- 将天气和场景作为 LLM 匹配的额外上下文输入

#### 4.2.5 AI 穿搭评分

**评分维度**

| 维度 | 英文键 | 含义 |
|------|--------|------|
| 色彩和谐 | colorHarmony | 颜色搭配是否和谐，对比色/互补色/同色系运用得当 |
| 风格统一 | styleCohesion | 上下装风格体系是否一致，混搭是否合理 |
| 时尚度 | trendiness | 是否符合当季流行趋势，有无设计感或潮流元素 |
| 实穿性 | practicality | 日常穿着是否方便舒适，场景适用范围是否广 |
| 创意度 | creativity | 搭配是否有巧思亮点，是否展现个人风格 |

**评分标准**
- 每项 1–100 分，五个维度独立评分
- 综合分 = 五维均值（四舍五入取整）
- 评分由多模态 LLM（Qwen-VL-Max）基于穿搭效果图视觉分析给出

**展示方式**
- 五维雷达图（Pentagon Radar Chart）
- 中心显示综合分，Hover 某一维度时高亮并显示该维度具体分数
- 雷达图下方显示 AI 搭配评语（2–3 句）
- 评分数据在推荐页、收藏页均展示（同款交互式雷达图组件）

**触发方式**
- 试穿页：对单套已生成穿搭点击「AI 评分」
- 推荐页：pipeline 自动批量评分；支持「重新打分」按钮手动触发
- 收藏页：自动展示已有评分数据，无需额外触发

### 4.3 P1 功能需求

#### 4.3.1 AI 穿搭顾问（对话式）

> 竞品参考：Cladwell "Ask Cladwell"（ChatGPT 接入）。OOTD 的差异化：基于用户**真实衣橱数据**回答，而非通用建议。

**核心能力**
- 用户以自然语言提问，LLM 结合衣橱单品库 + 天气数据回答
- 典型对话场景：
  - "这条白裙子配什么鞋好？" → 从衣橱中推荐具体单品
  - "我衣橱里有什么适合面试穿的？" → 按场合筛选 + 搭配建议
  - "下周去三亚出差 3 天，帮我规划穿搭" → 查目的地天气 + 逐日规划
  - "最近买什么能让衣橱搭配更多样？" → 衣橱缺口分析

**交互方式**
- 推荐页底部浮动入口，点击展开对话面板
- 支持引用衣橱中的具体单品（图片 + 名称）
- AI 回复中可内嵌单品卡片，点击可直接发起试穿

**技术方案**
- 复用现有 Qwen-Max LLM，将衣橱单品列表 + 天气作为 system prompt 上下文
- 流式输出（SSE），打字机效果展示

---

#### 4.3.2 快速穿搭（"随便穿穿"模式）

> 竞品参考：Whering "Dress Me"（随机穿搭）。OOTD 的差异化：不是纯随机，而是基于历史高分搭配 + 季节适配 + 加权随机的智能推荐。

**核心能力**
- 一键获取穿搭推荐，秒出结果，不触发完整 pipeline（不调试穿 API、不调评分、不调天气 API）
- 全部基于 SQLite 本地查询，复用历史缓存的试穿图和评分，零 API 调用

**推荐来源（按优先级 fallback）**

| 优先级 | 来源 | 条件 | 说明 |
|--------|------|------|------|
| 1 | 今日/目标日期已有推荐 | DailyRecommendation 表有当天记录 | 直接展示，零延迟 |
| 2 | 季节适配 + 历史高分 | Outfit.score ≥ 75 且关联 Item 的 season 匹配当前季节 | 加权随机选一套 |
| 3 | 历史高分兜底 | Outfit.score ≥ 70，无季节过滤 | 季节适配无结果时退化 |
| 4 | 任意已缓存试穿图 | Outfit 表有 resultImagePath | 新用户冷启动兜底 |

**季节适配策略（不调天气 API）**
- 用当前月份映射适合的 season 标签，零网络请求：
  - 3–5 月 → `["春", "四季"]`
  - 6–8 月 → `["夏", "四季"]`
  - 9–11 月 → `["秋", "四季"]`
  - 12–2 月 → `["冬", "四季"]`
- 查 Outfit 关联的 top/bottom Item，两件单品的 season 都匹配当前季节才纳入候选池

**加权随机策略**
- 权重 = `score - 阈值`（如 score=90 权重 15，score=76 权重 1，高分搭配被选中概率更高）
- 收藏的搭配额外加权：`isFavorite` 权重 ×1.5

**去重策略**
- 前端维护 `shownIds: Set<string>`（本次会话内已展示的 outfit ID）
- 每次从候选池中排除已展示的，避免连续点"换一套"看到重复
- 候选池耗尽时提示"已经看完所有推荐啦"并支持重置

**交互方式**
- 推荐页空状态下，除"生成推荐"按钮外，新增"随便穿穿"轻量入口
- 点击后卡片翻转动画，展示一套搭配（效果图 + 单品信息 + 历史评分雷达图）
- 底部按钮：「换一套」|「查看详情」

**联动价值**

**技术方案**
- 复用 qwen-vl-max，已有人像分析基础，增加色彩诊断专用 prompt
- 诊断结果结构化存储（JSON），与 PersonImage 关联
- 一次 API 调用完成诊断，成本极低

#### 4.3.11 穿搭广场（Show 场）— 冷启动增长引擎

> 核心洞察：新用户在上传衣橱之前无法感受产品核心价值（"看到自己穿上"的魔力），Show 场把 Aha 时刻从"分钟级"压缩到"秒级"——零衣橱即可体验试穿，体验驱动主动上传。

**产品目标**
- 解决冷启动问题：新用户打开即可看到效果、体验试穿，无需先上传衣橱
- 建立增长飞轮：看到好穿搭 → 一键试穿 → 上传自己的衣橱 → 产出优质内容 → 吸引更多用户
- 提升留存：每日可逛的新鲜内容，增加打开 App 的理由

**用户旅程**

```
新用户首次进入：
  → 打开 App，首页/底部导航有「广场」入口
  → 浏览他人分享的精美穿搭效果图（瀑布流）
  → 看到喜欢的搭配 → 点击「试穿同款」
  → 选择/上传自己的人像（引导完成人像建立）
  → AI 生成自己穿这套衣服的效果图（Aha 时刻！）
  → 觉得效果好 → 点击「加入我的衣橱」一键收录同款单品
  → 自然转化为活跃用户，开始上传自己的衣服

老用户分享：
  → 推荐页/收藏页生成满意的穿搭效果图
  → 点击「分享到广场」→ 可选添加配文
  → 收到他人试穿/点赞的通知 → 正向反馈 → 持续产出
```

**功能模块**

**A. 分享到 Show 场**
- 触发入口：推荐页、收藏页、试穿页的穿搭卡片上新增「分享到广场」按钮
- 分享内容：试穿效果图 + 搭配评分 + AI 评语 + 单品信息（名称、分类、图片）
- 可选配文：用户可添加一句分享感言
- **隐私保护**：只分享试穿效果图（AI 生成图），不暴露用户原始人像照片
- 分享后可随时撤回（从广场下架，不影响自己的收藏）

**B. 浏览广场**
- 瀑布流/卡片列表展示所有公开分享的穿搭
- 每张卡片展示：试穿效果图、综合评分、搭配单品缩略图、被试穿次数
- 排序方式：最新 / 最热（综合评分 × 试穿次数加权）/ 随机发现
- 支持按风格、场合、季节筛选
- 支持点赞

**C. 一键试穿同款**（核心差异化功能）
- 用户点击广场卡片上的「试穿同款」按钮
- 选择自己的人像（若未上传，引导先上传 → 完成人像建立冷启动）
- 系统取该搭配的原始单品图片（上衣 + 下装），调用现有 `generateTryon()` 生成试穿图
- 生成结果展示评分，默认加入收藏
- **技术实现**：完全复用现有 `lib/tryon.ts` 管线，零额外 AI 模型接入
- **缓存机制**：同一用户 + 同一单品组合自动缓存，不重复生成

**D. 一键加入衣橱**
- 广场卡片上的单品缩略图支持点击
- 展示单品详情（图片、分类、颜色、风格等属性）
- 点击「加入我的衣橱」→ 复制该 Item 的图片和属性，创建为当前用户的新单品
- 加入后可正常参与用户自己的搭配推荐

**数据模型（增量）**

```
ShowcasePost
  ├── id (cuid)
  ├── outfitId → Outfit           // 关联的穿搭效果
  ├── caption (可选)              // 用户配文
  ├── likes (默认 0)              // 点赞数
  ├── tryonCount (默认 0)         // 被试穿次数
  ├── isPublic (默认 true)        // 是否公开（撤回时设为 false）
  ├── createdAt
  └── updatedAt
```

> 注意：Item 和 Outfit 表无需修改，ShowcasePost 仅引用 outfitId，通过 Outfit 关联到 topItem / bottomItem 获取单品信息。

**冷启动内容策略**
- 团队提前用种子账号产出 20–30 套高质量穿搭内容
- 覆盖不同风格（休闲/通勤/约会/运动）和不同季节
- 标注真实的 AI 评分，建立质量基准

**成本分析**
- 浏览广场：零 API 成本（纯数据库查询 + 图片展示）
- 一键试穿：与普通试穿相同（¥0.2/次，受缓存机制保护）
- 一键加衣橱：零 API 成本（数据库复制）

**技术方案**
- 新增 Prisma Model `ShowcasePost`
- 新增 API Routes：`/api/showcase`（CRUD + 列表）、`/api/showcase/tryon`（试穿同款）
- 新增页面：`/showcase`（广场浏览页）
- 复用组件：穿搭卡片、雷达图评分、人像选择器
- 复用管线：`lib/tryon.ts` 的 `generateTryon()` 完全复用

### 4.4 P2 功能需求

#### 4.4.1 电商单品导入

> 竞品参考：搭搭（淘宝导入）、Whering（Chrome 插件）。冷启动是衣橱 App 最大痛点，此功能对用户增长至关重要。

- 备选：提供浏览器插件，在电商页面一键收录单品

#### 4.4.2 出行打包模式

> 竞品参考：Stylebook 打包清单。OOTD 已有多日天气预报 + LLM 匹配能力，自然延伸。

- 用户输入：目的地城市 + 出行天数 + 场合（出差/旅行/度假）
- 系统自动查询目的地多日天气预报
- LLM 基于天气 + 场合 + 衣橱单品，规划每日穿搭方案
- 输出打包清单（去重合并，标注每件单品在哪几天穿着）
- 支持手动调整后导出为图片/清单

#### 4.4.3 衣橱统计与 LLM 诊断

- 单品穿着次数统计
- Cost-per-wear 计算（购入价 ÷ 穿着次数）
- 衣橱构成分析（颜色分布、分类占比、品牌分布）
- 「僵尸单品」提醒（超过 60 天未穿着）
- **LLM 衣橱诊断**（差异化功能）：
  - 分析衣橱结构，发现搭配瓶颈（如"黑色上衣过多，缺少浅色下装"）
  - 基于现有单品推荐最高 ROI 的下一件购入建议
  - 季节更替提醒（如"入秋了，你只有 1 件外套，考虑补充"）

#### 4.4.4 穿搭 Before/After 对比

> 核心洞察：对比图是社交媒体上最有冲击力的内容形式，「换个搭法差这么多」的惊喜感驱动传播。

- 同一套单品，AI 生成两版试穿效果图：
  - 「随意穿」版：常规搭配方式，无特别搭配技巧
  - 「精心搭」版：加入搭配技巧（叠穿、塞衣角、配饰点缀、色彩呼应等）
- 左右滑动对比展示，中间分割线可拖动
- 两版各附 AI 评分 + 提升点总结
- 可一键生成 Before/After 分享图

**技术方案**
- 同一组合调两次试穿生成（不同 prompt 引导随意 vs 精心），API 成本 ×2
- 前端实现滑动对比组件（CSS clip-path 或 Canvas）

#### 4.4.5 衣橱万花筒（可视化）

> 核心洞察：把枯燥的衣橱数据变成赏心悦目的视觉作品，让「看自己的衣橱」变成一种享受。

- **色彩气泡图**：每件单品以主色调气泡展示，气泡大小 = 穿着频次，一眼看出色系偏好
- **色轮分布**：12 色相环上标注衣橱单品分布，发现色彩盲区
- **风格关键词云**：休闲 42% / 通勤 28% / 约会 15%，动态词云动画
- **类别甜甜圈图**：上衣/下装/外套/鞋子/配饰占比，中心显示总件数
- **季节覆盖雷达图**：春夏秋冬四季单品覆盖度，发现季节缺口

**技术方案**
- 纯前端实现（D3.js / Chart.js / 自定义 SVG），零 API 成本
- 数据来源：衣橱单品的已有标签（颜色、风格、季节、类别）
- 可生成「衣橱报告」分享图

#### 4.4.6 穿搭灵感 Moodboard

> 核心洞察：像翻时尚杂志一样浏览灵感，而不是被动接受推荐。

- AI 根据当季流行趋势 + 用户衣橱 + 个人色彩季型，生成视觉灵感板
- Moodboard 内容：
  - 单品抠图拼贴
  - 从单品提取的色彩方案（3-5 色）
  - 风格关键词标签
  - 关联的试穿效果图
  - 搭配建议文案
- 每周/每季自动更新，呈现当季专属灵感
- 瀑布流/杂志版式排列，支持长按保存

**技术方案**
- LLM 生成灵感文案 + 搭配方案，前端 CSS Grid 排版渲染
- 复用已有单品图 + 试穿图，零额外图像生成成本

### 5.2 隐私与安全

- 用户人像照片：仅用于生成试穿效果图，不上传至公共服务器，不用于模型训练，不对第三方开放
- 衣橱数据：加密存储，用户可随时导出或删除全部数据

## 七、技术架构概述

### 7.1 MVP 开发阶段（当前）

使用 Next.js 全栈 Web App 快速验证核心功能，正式上线后迁移至 iOS 原生客户端。

```
Next.js Web App（App Router + TypeScript + Tailwind CSS）
  ├── 衣橱管理页面
  ├── 人像管理页面
  ├── 试穿生成页面
  ├── 每日推荐页面（SSE 流式进度）
  └── 收藏管理页面

API Routes（Next.js Server-side）
  ├── 衣橱服务（单品 CRUD）
  ├── 人像服务（CRUD + 默认人像）
  ├── 试穿服务（调用 DashScope + 缓存）
  ├── 推荐服务（LLM 匹配 + 批量试穿 + 评分 pipeline）
  └── 评分服务（多模态 LLM 五维评分）

AI 服务（DashScope / 阿里云百炼）
  ├── qwen-image-edit    — 单品自动抠图去背景（去除背景 + 人物模特）
  ├── qwen-image-edit-max — 人像优化（光影均匀、构图裁剪、背景清理、超分辨率，1536×2048 高清输出）
  ├── qwen-image-2.0-pro — 虚拟试穿图像生成
  ├── qwen-vl-max        — 多模态穿搭评分（五维雷达图）+ AI 自动识别分类
  └── qwen-max           — 智能搭配组合推荐

数据存储
  ├── Prisma 7 + SQLite（本地文件数据库）
  └── 图片资源：本地文件系统（public/uploads/）
```

## 八、竞品差异化总结

| 维度 | 竞品现状 | 本产品 |
|------|---------|--------|
| 搭配呈现 | 单品清单 / 拼图组合 | **AIGC 生成用户真实穿着效果图** |
| 虚拟试穿 | 无 或 效果粗糙 | **高质量扩散模型，效果接近真实** |
| 推荐个性化 | 基于通用搭配规则 | **基于用户真实衣橱 + 天气 + 场景** |
| 数据隐私 | 人像数据不透明 | **人像不上公共服务器，明确隐私承诺** |
| 变现模式 | 导购佣金（影响体验） | **订阅制工具，不打扰用户** |

## 九、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 试穿效果图质量达不到预期 | 中 | 高 | MVP 阶段先做上半身试穿，全身方案待模型成熟后迭代 |
| 生成速度慢影响体验 | 中 | 高 | 每日推荐提前夜间预生成，用户打开即可查看 |
| 用户上传衣物流失（冷启动麻烦） | 高 | 中 | 穿搭广场（Show 场）：新用户零衣橱即可一键试穿他人搭配，体验驱动上传；降低门槛至 5 件即可推荐 |
| AI 识别分类错误率高 | 中 | 低 | 识别结果可手动修正，持续收集标注数据优化 |
| 人像隐私顾虑导致用户不上传 | 中 | 高 | 提供「不上传人像」模式（使用通用模特替代生成） |
