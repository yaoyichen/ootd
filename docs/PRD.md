# PRD：每日穿搭 AI 应用（OOTD）

**版本**：v0.1
**日期**：2026-03-14
**状态**：草稿

---

## 一、产品概述

### 1.1 产品背景

当前市面上的穿搭类应用（Stylebook、Acloset、搭搭等）核心能力停留在「搭配清单推荐」层面，无法让用户直观看到自己穿上某套衣服的实际效果。用户每天面对衣橱时仍然陷入「有很多衣服却不知道穿什么」的困境。

AIGC 图像生成技术（虚拟试穿模型）的成熟，为这个问题提供了全新的解决路径。

### 1.2 产品定位

> **一个能「看到自己穿上效果」的 AI 每日穿搭助手**

用户上传自己的衣物和个人照片，App 通过 AIGC 生成用户真实穿着任意搭配组合的效果图，并结合每日天气和出行场景，智能推荐当天最合适的穿搭方案。

### 1.3 核心价值主张

- **所见即所得**：不是「这件配那件」的文字推荐，而是生成你穿上效果的真实图像
- **懂你的衣橱**：归档你真实拥有的衣物，而非通用搭配模板
- **每日新鲜感**：结合天气 + 场景，每天打开都有专属推荐

## 二、目标用户

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| **年龄** | 18–35 岁 |
| **性别** | 以女性为主，兼顾男性 |
| **特征** | 有一定衣物积累，有穿搭意识，但日常选衣耗时长 |
| **设备** | iOS / Android 智能手机 |
| **场景** | 每天早晨出门前、购物前规划搭配、记录当日 OOTD |

### 2.2 用户痛点

1. **选择困难**：衣橱里衣服不少，每天早上却不知道穿什么
2. **试穿成本高**：想知道搭配效果必须真实穿上，费时费力
3. **缺乏灵感**：翻来覆去就那几套，忘记自己有哪些衣服
4. **场景错配**：穿了不合时宜的衣服（天气/场合判断失误）

### 2.3 用户旅程

```
早晨起床
  → 打开 App，看到「今日推荐」（基于天气 + 今日行程）
  → 浏览 3 套 AI 生成的穿搭效果图（自己穿着自己衣服）
  → 选择一套，或微调后确认
  → 出门
  → 可选：拍照打卡 OOTD，记录当日穿搭
```

## 三、产品主链路

```
【Step 1】建立衣橱
  用户拍照 / 上传单品图片
      ↓
  AI 自动抠图（去背景）
      ↓
  AI 识别分类（上衣 / 下装 / 外套 / 鞋子 / 配饰）
      ↓
  自动打标（颜色 / 风格 / 季节 / 场合）
      ↓
  单品入库，衣橱归档完成

【Step 2】建立个人形象
  用户上传全身人像照片（正面，光线清晰）
      ↓
  系统解析人体姿态 / 体型信息
      ↓
  作为后续试穿生成的基准人像

【Step 3】AIGC 穿搭生成
  从衣橱中选取单品组合（手动 or AI 推荐）
      ↓
  调用虚拟试穿模型（IDM-VTON / Fashn.ai API）
      ↓
  生成用户穿上该组合的真实效果图
      ↓
  展示给用户，支持重新生成 / 微调

【Step 4】每日推荐
  每日打开 App
      ↓
  读取实时天气（温度 / 天气状况 / 降水概率）
      ↓
  读取用户今日场景（可手动选择：上班 / 约会 / 运动 / 休闲 / 出行）
      ↓
  AI 从衣橱中筛选合适单品，生成 3 套推荐搭配
      ↓
  调用试穿模型，生成每套方案的效果图
      ↓
  推送 / 展示「今日穿搭推荐」
```

### 4.2 P0 功能详细需求

#### 4.2.1 衣橱管理

**单品上传**
- 支持拍照上传（调用系统相机）
- 支持从相册批量选择上传（最多 20 张/次）
- 上传后 AI 自动去背景，输出透明底单品图
- AI 自动识别并填写：分类、主色调、风格、适合季节
- 用户可手动修改所有自动识别字段
- 支持录入：品牌、购入价格、购入日期、备注

**单品分类体系**
```
服装
  ├── 上衣（T恤 / 衬衫 / 针织 / 卫衣 / 背心）
  ├── 下装（裤子 / 裙子 / 短裤）
  ├── 外套（夹克 / 大衣 / 风衣 / 西装）
  └── 连体（连衣裙 / 连体裤 / 套装）
鞋子
  ├── 运动鞋 / 板鞋
  ├── 皮鞋 / 正装鞋
  ├── 凉鞋 / 拖鞋
  └── 靴子
配饰
  ├── 包袋
  ├── 帽子
  ├── 围巾 / 披肩
  └── 首饰 / 腰带 / 眼镜
```

**衣橱管理操作**
- 支持按分类、颜色、季节、场合多维筛选
- 支持搜索单品（关键词 / 品牌）
- 支持标记「闲置」「借出」「捐赠」状态
- 支持删除单品（二次确认）

---

#### 4.2.2 个人形象

- 用户上传全身正面照片（引导提示：光线均匀、背景简洁、全身入镜）
- 支持上传多张人像（不同姿态 / 场景备用）
- 设置「默认人像」，用于每日推荐生成
- 人像仅存储在用户本地 / 私有云，不对外展示，不用于模型训练

---

#### 4.2.3 AIGC 试穿生成

**触发方式**
- 用户在衣橱中选择单品后，点击「试穿」
- 用户在推荐页选择搭配方案后，系统自动生成效果图

**生成参数**
- 基准人像：使用用户设置的默认人像
- 单品组合：上衣 + 下装（必选），外套 / 鞋子 / 配饰（可选）
- 生成数量：每次生成 1 张，可点击「重新生成」

**生成体验**
- 生成等待时间目标：≤ 15 秒
- 等待期间展示加载动画，不可进行其他操作（避免重复请求）
- 生成失败时给出明确提示并提供重试入口

**图片处理**
- 生成图默认保存至「我的搭配」收藏夹
- 支持保存到系统相册
- 支持分享（微信、小红书等）

**技术方案（MVP 阶段）**
- 优先集成 Fashn.ai API（商业 API，稳定，快速上线）
- 备选：OOTDiffusion / IDM-VTON 自托管（成本更低，需 GPU 资源）
- 生成分辨率：768×1024 px

---

#### 4.2.4 每日推荐

**触发时机**
- 用户每日首次打开 App 时，自动生成当日推荐
- 用户可手动刷新推荐

**推荐逻辑**

```
输入：
  - 当日天气（温度范围、天气状况、降水概率）
  - 用户选择的今日场景（上班 / 约会 / 运动 / 休闲 / 出行）
  - 用户衣橱中的所有单品

筛选规则：
  1. 按温度筛选适合季节的单品
  2. 按天气状况过滤（雨天排除浅色/不防水鞋子）
  3. 按场景筛选风格匹配的单品
  4. 排除最近 7 天已穿着的搭配（避免重复）

搭配生成：
  - 上衣 + 下装为核心组合
  - AI 按色彩协调原则（同色系 / 互补色 / 中性色搭配）选取组合
  - 生成 3 套候选方案

输出：
  - 3 套搭配方案的虚拟试穿效果图
  - 每套方案附简短说明（例：「今天 12°C 有雨，这套休闲又防雨」）
```

**天气数据源**
- 接入和风天气 API 或高德天气 API
- 基于用户授权的 GPS 定位获取当地天气
- 用户可手动切换城市

**场景选择**
- 每日推荐页顶部展示场景选择器
- 场景选项：上班、约会、休闲、运动、出行、正式场合
- 默认根据日历（工作日 → 上班，周末 → 休闲）预选

### 4.3 P1 功能需求

#### 4.3.1 OOTD 记录

- 用户确认当日穿搭后，支持拍照记录实际穿着效果
- 日历视图展示历史 OOTD（月视图，每天一张缩略图）
- 支持为 OOTD 添加心情标签和备注
- 自动关联已穿着的单品，更新穿着频次统计

#### 4.3.2 搭配管理

- 「我的搭配」：收藏所有历史生成的搭配效果图
- 支持手动创建搭配（从衣橱拖拽单品组合，然后生成试穿图）
- 支持将搭配与场合、季节关联，便于后续筛选复用

---

### 4.4 P2 功能需求

#### 4.4.1 衣橱统计

- 单品穿着次数统计
- Cost-per-wear 计算（购入价 ÷ 穿着次数）
- 衣橱构成分析（颜色分布、分类占比、品牌分布）
- 「僵尸单品」提醒（超过 60 天未穿着）

#### 4.4.2 社区分享

- 用户可选择公开发布 OOTD（默认不公开）
- 发现页：按风格、场合、季节浏览他人 OOTD
- 支持点赞、收藏他人搭配
- 支持将他人搭配中的单品加入「想要清单」

## 五、非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 试穿图生成时间 | ≤ 15 秒（P90） |
| App 冷启动时间 | ≤ 2 秒 |
| 图片上传响应 | ≤ 3 秒（单张，4G 网络） |
| 每日推荐加载 | ≤ 5 秒（含生成） |

### 5.2 隐私与安全

- 用户人像照片：仅用于生成试穿效果图，不上传至公共服务器，不用于模型训练，不对第三方开放
- 衣橱数据：加密存储，用户可随时导出或删除全部数据
- 生成的试穿图：默认私有，用户主动操作才可分享

### 5.3 平台支持

- **MVP 阶段**：iOS 优先（iPhone，iOS 16+）
- **后续**：Android

| 包含 | 不包含 |

| 指标 | 目标值 | 验证方式 |

## 七、技术架构概述

```
客户端（iOS）
  ├── 衣橱管理模块
  ├── 每日推荐模块
  ├── AIGC 试穿模块（调用后端 API）
  └── 本地数据缓存（Core Data / SQLite）

后端服务
  ├── 用户服务（账号 / 数据同步）
  ├── 衣橱服务（单品 CRUD / 分类管理）
  ├── AI 识别服务（抠图 / 单品分类）
  │     └── 调用：阿里云图像识别 / 自研模型
  ├── 天气服务（接入和风天气 API）
  ├── 推荐服务（搭配逻辑 / 规则引擎）
  └── AIGC 生成服务
        └── 调用：Fashn.ai API（MVP）/ OOTDiffusion 自托管（后续）

存储
  ├── 用户数据：云数据库（加密）
  ├── 图片资源：对象存储（OSS）
  └── 人像数据：用户私有存储（不落公共服务器）
```

## 八、竞品差异化总结

| 维度 | 竞品现状 | 本产品 |
|------|---------|--------|
| 搭配呈现 | 单品清单 / 拼图组合 | **AIGC 生成用户真实穿着效果图** |
| 虚拟试穿 | 无 或 效果粗糙 | **高质量扩散模型，效果接近真实** |
| 推荐个性化 | 基于通用搭配规则 | **基于用户真实衣橱 + 天气 + 场景** |
| 数据隐私 | 人像数据不透明 | **人像不上公共服务器，明确隐私承诺** |
| 变现模式 | 导购佣金（影响体验） | **订阅制工具，不打扰用户** |

## 九、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 试穿效果图质量达不到预期 | 中 | 高 | MVP 阶段先做上半身试穿，全身方案待模型成熟后迭代 |
| 生成速度慢影响体验 | 中 | 高 | 每日推荐提前夜间预生成，用户打开即可查看 |
| 用户上传衣物流失（冷启动麻烦） | 高 | 中 | 降低初始门槛：只需上传 5 件即可体验推荐 |
| AI 识别分类错误率高 | 中 | 低 | 识别结果可手动修正，持续收集标注数据优化 |
| 人像隐私顾虑导致用户不上传 | 中 | 高 | 提供「不上传人像」模式（使用通用模特替代生成） |

## 十、迭代规划

### Phase 1 — MVP（试穿验证）
- 单品上传 + AI 识别分类
- 个人人像上传
- 上衣/下装虚拟试穿效果图生成
- 天气联动每日推荐

### Phase 2 — 完整体验
- 多件叠穿（外套 + 鞋子 + 配饰）
- OOTD 打卡记录
- 衣橱统计（cost-per-wear）
- 行程日历联动

### Phase 3 — 增长
- 社区 OOTD 发布与发现
- 电商单品导入（淘宝 / 小红书）
- Android 版本
- 购物推荐（衣橱缺口分析）
