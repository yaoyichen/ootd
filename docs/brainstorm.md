# 每日穿搭应用 - 脑暴

## 核心功能方向

### 1. 衣橱管理
- 拍照/上传单品，自动识别类别（上衣、裤子、鞋子等）
- 标签系统：颜色、风格、场合、季节
- 单品使用频率统计，发现"僵尸衣物"

### 2. 穿搭生成
- 根据天气自动推荐当日穿搭
- 根据日历事件推荐（面试、约会、健身）
- AI 智能搭配，基于色彩/风格协调规则
- 用户手动拖拽组合

### 3. OOTD 记录
- 每日打卡，拍照记录当天穿搭
- 日历视图回顾历史穿搭
- 避免短期内重复穿搭

### 4. 社区 & 灵感
- 发现页：浏览他人穿搭
- 按场合/风格/体型筛选参考
- 收藏灵感板（类似 Pinterest）

## 差异化亮点方向

| 方向 | 想法 |
|------|------|
| **天气联动** | 接实时天气，"今天18°C 有雨，推荐这套" |
| **行程感知** | 接日历，周一开会 vs 周六逛街推荐不同 |
| **胶囊衣橱** | 引导用户建立少而精的衣橱 |
| **AI 试穿** | 上传人像，虚拟试穿效果 |
| **购物建议** | 分析衣橱缺口，推荐补充单品 |
| **情绪穿搭** | 今天想要什么感觉？自信/舒适/低调 |

## 核心产品愿景（重点方向）

### 产品主链路

```
1. 拍照上传单品（衣服/裤子/鞋子/配饰）
       ↓ AI 自动识别分类 + 抠图归档
2. 上传个人照片（全身人像）
       ↓ AIGC 虚拟试穿 → 生成任意搭配组合的效果图
3. 每日打开 App
       ↓ 读取当天天气 + 用户外出需求/场景
       ↓ 生成今日推荐搭配方案（含真实效果图）
```

### 与竞品的核心差异

现有竞品（Acloset、搭搭、Cladwell 等）只能给出「搭配清单」，
**本产品直接生成用户穿上自己衣服的效果图** — 所见即所得。

### 技术实现方向

| 方案 | 技术路径 | 优劣 |
|------|---------|------|
| **Try-on 模型（首选）** | IDM-VTON、CatVTON、OOTDiffusion | 效果真实，上半身成熟，全身在迭代 |
| **商业 API** | Fashn.ai、可图 Klingai | 快速集成，有成本，效果稳定 |
| **图像编辑** | ControlNet + IP-Adapter | 灵活但一致性难控制 |

### MVP 验证优先级

1. **先跑通试穿生成效果** — 验证单件上衣/裤子穿在用户身上的图像质量
2. 再做衣橱归档（拍照识别 + 分类）
3. 最后接入天气/场景推荐逻辑

### 主要技术挑战

- **试穿一致性**：多次生成需保持人脸/体型稳定
- **多件叠穿**：上衣 + 裤子 + 鞋同时上身（当前模型弱点）
- **生成速度**：每日推荐需秒级出图，冷启动延迟是体验杀手
- **细节还原**：logo、面料纹理、颜色准确性

## 几个可能的产品定位

1. **极简工具型** — 快速记录 + 天气推荐，轻量高频使用
2. **社区分享型** — 以 OOTD 打卡和灵感发现为核心
3. **AI 造型师型** — 深度分析衣橱，提供专业搭配建议
4. **胶囊衣橱型** — 帮用户建立更有品质感的精简衣橱
