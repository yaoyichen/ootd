# 电商单品导入 — 技术方案

**更新日期**：2026-03-17
**优先级**：P2（Phase 2）
**状态**：方案调研完成，待排期

---

## 一、需求背景

冷启动是衣橱类 APP 最大痛点。用户手动拍照上传每件单品耗时长、门槛高，导致大量用户在建立衣橱阶段流失。竞品（搭搭、今日装、Whering）均已提供不同形式的电商导入能力。

**目标**：让用户能将淘宝/天猫已购的服装单品快速导入衣橱，复用现有 AI 抠图 + 自动识别能力，最大程度减少手动录入。

## 三、可选技术方案

### 方案 A：淘口令 / 商品链接解析（推荐，MVP 首选）

**原理**：用户复制淘宝商品链接或淘口令，APP 解析后获取商品详情。

**流程**：

```
用户复制淘口令（￥xxx￥）或商品链接
    ↓
粘贴到 APP 输入框（Web 端）/ 剪贴板检测（原生 APP 端）
    ↓
后端解析淘口令 → 获取商品 ID + 真实 URL
    ↓
调用商品详情 API → 获取标题、主图、价格、品牌、属性
    ↓
主图自动抠图（复用现有 qwen-image-edit 能力）
    ↓
AI 自动识别分类（复用现有 Qwen-VL 识别能力）
    ↓
自动填充到单品录入表单 → 用户确认后保存
```

**淘口令解析方式**：

| 方式 | 说明 | 成本 |
|------|------|------|
| 阿里妈妈官方 SDK | `WirelessShareTpwdQueryRequest` 接口 | 免费，需淘宝联盟开发者账号 |
| 第三方解析 API | 如订单侠 `api.tbk.dingdanxia.com/tkl/query` | 按量付费，约 ¥0.001/次 |
| 自行解析 | 正则提取短链接 → HTTP 302 跟踪 → 提取商品 ID | 免费，但不稳定，淘宝反爬频繁变更 |

**优点**：
- 用户操作简单（复制 → 粘贴 → 确认）
- 技术成熟，实现成本低
- 能获取结构化数据（标题、价格、品牌）

**缺点**：
- 依赖第三方 API 或淘宝联盟账号
- 一次只能导入一件，无法批量
- 合规性需关注（淘宝反爬政策）

---

### 方案 B：商品截图 AI 识别（兜底方案）

**原理**：用户截图商品页面，AI 视觉模型识别图中信息。

**流程**：

```
用户截图淘宝/小红书商品页面
    ↓
上传截图到 APP
    ↓
Qwen-VL 识别截图中的：商品名称、主图区域、价格、品牌
    ↓
提取主图 → 自动抠图
    ↓
自动填充属性 → 用户确认保存
```

**优点**：
- 零第三方依赖，不受反爬影响
- 支持任意电商平台（淘宝/京东/小红书/得物等）
- 合规无风险

**缺点**：
- 识别精度不如结构化 API 数据
- 用户操作多一步（需要截图）
- 截图质量影响识别效果

---

### 方案 C：淘宝开放平台 OAuth 批量导入（理想方案，门槛高）

**原理**：用户 OAuth 授权后，通过淘宝开放平台 API 读取已购订单。

**流程**：

```
用户点击「导入淘宝订单」
    ↓
跳转淘宝 OAuth 授权页 → 用户授权
    ↓
获取 access_token（有效期 1 年）
    ↓
调用订单 API → 筛选服装类目订单
    ↓
批量获取商品主图 + 属性
    ↓
逐件抠图 + 识别 → 批量入库
    ↓
用户在衣橱中确认/修改
```

**优点**：
- 体验最好，一键批量导入
- 数据最准确（直接读取订单结构化数据）

**缺点**：
- 淘宝开放平台审批严格，需企业资质
- 个人消费者订单数据权限几乎批不下来
- 需要在聚石塔安全云环境中部署（增加运维成本）
- 授权流程重，用户信任成本高

---

### 方案 D：浏览器插件（PC 端补充）

**原理**：Chrome/Safari 扩展，在电商页面一键收录商品。

**适用场景**：PC 端浏览电商时顺手收录

**优先级低**：国内用户以手机购物为主，PC 端场景有限

## 四、推荐实施策略

分阶段实施，从轻到重：

### 第一步：淘口令解析（轻量，快速上线）

- 前端：单品上传页新增「从链接导入」入口，支持粘贴淘宝链接/淘口令
- 后端：新增 `/api/import/taobao` 接口，解析链接 → 获取商品详情 → 返回图片和属性
- 复用：现有 AI 抠图 + 自动识别流程
- 依赖：申请淘宝联盟开发者账号 或 接入第三方解析 API

### 第二步：截图识别（兜底 + 多平台）

- 前端：单品上传页新增「从截图导入」入口
- 后端：Qwen-VL 识别截图 → 提取商品信息 + 主图
- 优势：覆盖小红书、京东、得物等非淘宝平台

### 第三步（可选）：OAuth 批量导入

- 条件：产品正式上线、有企业资质后评估
- 取决于淘宝开放平台审批结果

## 五、技术要点

### 淘口令格式识别

```typescript
// 淘口令正则匹配
const TKL_REGEX = /[￥$]([a-zA-Z0-9]+)[￥$]/;

// 淘宝短链接匹配
const TB_SHORT_REGEX = /https?:\/\/(m\.tb\.cn|s\.click\.taobao\.com)\/\S+/;

// 淘宝商品链接匹配
const TB_ITEM_REGEX = /https?:\/\/.*?(taobao|tmall)\.com\/.*?id=(\d+)/;
```

### 商品详情需要提取的字段

| 字段 | 映射到衣橱 Item | 说明 |
|------|----------------|------|
| 商品标题 | name | 可能需要 AI 精简 |
| 商品主图 | imagePath | 按优先级选图：白底图 → SKU 图 → AI 评估（详见图片选择策略） |
| 价格 | price（待扩展） | 购入价格 |
| 品牌 | brand（待扩展） | 从标题或属性中提取 |
| 颜色/尺码 | color | 从 SKU 属性中提取 |

### 与现有流程的集成点

```
导入的商品图片
    ↓
复用 POST /api/items 现有上传流程
    ↓
自动抠图（qwen-image-edit）→ 已有
    ↓
自动识别分类（Qwen-VL）→ 已有
    ↓
SHA-256 去重检测 → 已有
    ↓
保存到衣橱 → 已有
```

核心优势：导入功能只需要做「获取商品图片 + 属性」这一层，后续全部复用现有 pipeline。

### 商品图片选择策略（核心难点）

淘宝商品页面上的图片种类繁多，直接抓取容易拿到带水印、拼图、模特场景图等「脏」图。需要精准选出最适合衣橱收录的**干净单品图**。

#### 淘宝商品图片类型

| 图片类型 | 数量 | 典型特征 | 适用性 |
|----------|------|----------|--------|
| 主图 1-4 | 4 张 | 模特上身图、场景图、细节图，可能有营销文字/水印 | ⚠️ 中等 — 需筛选 |
| **主图 5（白底图）** | **1 张** | **淘宝强制要求：纯白背景、无文字水印、商品居中、占比 ≥ 60%** | **✅ 最佳选择** |
| SKU 属性图 | N 张 | 通常为纯白/浅灰背景的平铺图，每个颜色一张 | ✅ 优秀备选 |
| 详情图 | 多张 | 长图拼接、营销图文混排 | ❌ 不适用 |

> **关键发现**：淘宝/天猫强制要求商家上传的**第 5 张主图必须为白底图**（800×800 以上，纯白背景，无文字/水印/拼接），这是平台搜索推荐的硬性规定。这张图天然就是最适合衣橱导入的图片。

#### 图片选择优先级算法

```
优先级 1：第 5 张主图（白底图）
    ↓ 命中率高，淘宝/天猫强制要求
    ↓ 直接使用，可跳过抠图步骤（已是白底）

优先级 2：SKU 属性图（匹配用户购买的颜色/尺码）
    ↓ 通常为干净的平铺白底图
    ↓ 可直接使用或轻度抠图

优先级 3：AI 评估主图 1-4
    ↓ 调用 Qwen-VL 逐张评估图片「干净度」
    ↓ 选出背景最简洁、无文字水印的那张
    ↓ 经过抠图处理后使用

兜底：取主图第 1 张 → 强制抠图
```

#### 通过 API 获取第 5 张白底图

商品详情 API 返回的 `item_imgs` 数组中，图片按上传顺序排列，第 5 张（index=4）即为白底图：

```typescript
interface TaobaoItemDetail {
  item_imgs: Array<{ url: string; position: number }>;
  prop_imgs: Array<{ url: string; properties: string }>; // SKU 图
  // ...
}

function selectBestImage(detail: TaobaoItemDetail): {
  url: string;
  source: 'white_bg' | 'sku' | 'main' | 'fallback';
  needsSegmentation: boolean;
} {
  const mainImgs = detail.item_imgs;
  const skuImgs = detail.prop_imgs;

  // 优先级 1：第 5 张主图（白底图）
  if (mainImgs.length >= 5) {
    return {
      url: mainImgs[4].url,
      source: 'white_bg',
      needsSegmentation: false, // 已是白底，通常无需抠图
    };
  }

  // 优先级 2：SKU 属性图
  if (skuImgs.length > 0) {
    return {
      url: skuImgs[0].url,
      source: 'sku',
      needsSegmentation: false, // SKU 图通常也是白底
    };
  }

  // 优先级 3 / 兜底：主图第 1 张
  return {
    url: mainImgs[0]?.url ?? '',
    source: mainImgs.length > 1 ? 'main' : 'fallback',
    needsSegmentation: true, // 需要抠图处理
  };
}
```

#### AI 辅助图片质量评估（优先级 3 细化）

当白底图和 SKU 图都不可用时，调用 Qwen-VL 对主图 1-4 进行「干净度」评分：

```typescript
const IMAGE_QUALITY_PROMPT = `请评估这张商品图片是否适合作为衣橱单品图：
评分维度（0-10）：
1. 背景干净度：纯色/简洁背景得高分，杂乱场景得低分
2. 商品完整度：单品完整展示得高分，局部特写得低分
3. 无干扰元素：无文字水印/价格标签/拼图得高分
4. 商品占比：商品占画面主体得高分

返回 JSON: { "score": 总分, "reason": "一句话理由" }`;
```

当主图 1-4 中有多张可选时，逐张评估取最高分。考虑到 API 调用成本，可设置阈值：评分 ≥ 7 即选用，无需评估所有图片。

#### 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 商品不足 5 张主图（无白底图） | 降级到 SKU 图 → AI 评估 → 兜底抠图 |
| 白底图上仍有轻微水印 | 进入抠图流程清除（现有能力可覆盖） |
| 用户购买了特定颜色 | 优先匹配该颜色的 SKU 图 |
| 非标品（如围巾、配饰） | 白底图规则同样适用，淘宝规则不区分品类 |
| 截图导入（方案 B） | 无结构化数据，完全依赖 AI 评估 + 抠图 |

#### 抠图流程优化

根据图片来源决定是否需要抠图，减少不必要的 API 调用：

```
白底图 / SKU 图 → 检测背景是否已为纯白
    ├─ 是 → 跳过抠图，直接保存（节省 ~¥0.04/次）
    └─ 否 → 走现有抠图流程

模特上身图 / 场景图 → 必须抠图
```

## 六、TAOBAO_API_KEY 获取与配置

本项目使用 **订单侠（dingdanxia.com）** 作为淘口令解析和商品详情的第三方 API 服务。以下是完整的注册、获取 API Key 和配置步骤。

### 1. 注册订单侠账号

1. 打开 [订单侠官网](https://www.dingdanxia.com/)，点击右上角「注册」
2. 使用手机号注册并完成短信验证
3. 登录后进入用户中心

### 2. 获取 API Key

1. 登录后进入 [个人中心 → API 管理](https://www.dingdanxia.com/user/api) 页面
2. 页面上会显示你的 **apikey**（一串字母数字组合），复制保存
3. 如果没有 apikey，点击「生成 API Key」按钮创建

### 3. 充值（按量付费）

订单侠 API 按调用次数计费，需要预充值：

1. 进入 [充值页面](https://www.dingdanxia.com/user/recharge)
2. 选择充值金额（最低 10 元起）
3. 支持支付宝/微信支付

**费用参考**：

| API 接口 | 单次费用 | 说明 |
|----------|---------|------|
| 淘口令解析 `/tkl/query` | ~¥0.001 | 解析淘口令获取商品 ID |
| 商品详情 `/item/detail` | ~¥0.001 | 获取商品标题、图片、价格等 |

> 每次淘宝导入调用 2 个接口，总成本约 **¥0.002/次**。充值 10 元可用约 5000 次。

### 4. 配置环境变量

将获取到的 apikey 配置到项目环境变量中：

```bash
# 在 web/.env 文件中添加（如果没有 .env 文件，从 .env.example 复制一份）
cp web/.env.example web/.env

# 编辑 web/.env，找到以下行并填入你的 apikey
TAOBAO_API_KEY=your-dingdanxia-api-key
```

将 `your-dingdanxia-api-key` 替换为你在订单侠后台复制的真实 apikey。

### 5. 验证配置

配置完成后，启动开发服务器并测试：

1. `cd web && npm run dev`
2. 打开 `http://localhost:3000/wardrobe/add`
3. 切换到「淘宝导入」tab
4. 粘贴一个淘口令（如 `￥testABC￥`）或商品链接，点击「导入商品」
5. 如果配置正确，应能看到商品图片和属性自动填充到表单中

### 6. 常见问题

| 问题 | 原因 | 解决方式 |
|------|------|----------|
| 提示「TAOBAO_API_KEY not configured」 | 未配置环境变量 | 检查 `web/.env` 中是否有 `TAOBAO_API_KEY=xxx` |
| 提示「淘口令解析失败」 | apikey 无效或余额不足 | 登录订单侠后台检查 apikey 是否正确、余额是否充足 |
| 提示「获取商品详情失败」 | 商品已下架或链接无效 | 换一个在售商品的链接重试 |
| 提示「图片下载失败」 | 淘宝 CDN 限制 | 一般为临时问题，稍后重试 |
| 修改 `.env` 后没生效 | Next.js 需要重启 | 停止开发服务器后重新 `npm run dev` |

### 7. 使用的 API 接口文档

本项目调用了订单侠的两个接口：

- **淘口令解析**：[文档](https://www.dingdanxia.com/doc/5/8) — `GET /tkl/query?apikey=xxx&tkl=￥abc￥`
- **商品详情**：[文档](https://www.dingdanxia.com/doc/5/1) — `GET /item/detail?apikey=xxx&item_id=123456`
